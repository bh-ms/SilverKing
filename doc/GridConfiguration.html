The GridConfiguration Framework

Overview

The GridConfiguration Framework simplifies configuration settings for SilverKing (and other distributed applications) by providing a single identifier that groups configuration settings. A GridConfiguration provides a mapping from a user-specified identifier to a group of environment variables.

Every SilverKing instance needs three configuration settings to run: the name of the SilverKing instance (e.g. SK.My.Instance), the port that SK accepts client connections on (e.g. 7777), and the ZooKeeper ensemble definition (e.g. zk1:9999,zk2:9999). While it is possible to specify these manually, SilverKing generally uses a grid configuration in place of these (e.g. GC_SK_My_Instance).

Defining Grid Configurations

For simple setups, StaticInstanceCreator.sh will automatically create a grid configuration file when it is run.

SilverKing expects the environment variable GC_DEFAULT_BASE to point to the location where it can find the grid configuration files. For configurations created by StaticInstanceCreator.sh, this is the path used for -G. For manual configurations, this path is the location of the .env files. For example: 

GC_DEFAULT_BASE=/my/gc/path
