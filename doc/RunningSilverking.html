Running SilverKing

Overview

This section describes several ways to run SilverKing starting with simple single-server setups, and progressing to large-scale setups that require health monitoring and dynamic topology changes.

Standalone Server

The simplest way to run SilverKing is to start a standalone server from the command prompt. This is done as follows: 

./skc -s __EMBEDDED_KVS__
This starts a SilverKing client shell with a server embedded in it.

Embedded in a Client

Another simple way to run SilverKing is to run an embedded instance inside of the same Java VM that the client resides in. This is useful for testing purposes. Data in an embedded instance will not persist after the VM exits. See the source code examples for more details on this method. (At present, starting up the embedded SK instance takes several seconds.)

Static IP List

The simplest way to configure a distributed SK instance is using a static list of IP addresses. Given such a list, the StaticInstanceCreator.sh script will populate all configuration required to run such an instance.

For example, to create a simple SK configuration we can run: 

./StaticInstanceCreator.sh -G /tmp -d SK.MyTest -s  -r 1 -z zkserver:9981 -g GC_SK_MY_TEST

Full-featured Configuration

A fully-featured distributed configuration enables powerful capabilities such as the ability to tolerate server failures, change topology, replication level, and data distribution on the fly. See FullScaleAdministration.html for instructions on administering a full-scale SilverKing installation.
